How It Works

Three steps to mistake-proof vibe coding

1

Feedback

Give 👍 or 👎 on your AI agent's actions. Feedback is stored in a SQLite+FTS5 lesson DB. On 👎, get corrective actions from similar past failures. Example: you 👎 a risky migration → it auto-promotes to a "never run DROP on prod" gate.

2

Rules

Repeated failures auto-promote into prevention rules. Thompson Sampling adapts which rules fire — high-risk patterns get stricter enforcement while low-risk ones stay relaxed.

3

Gates

Rules become Pre-Action Gates that block your agent before it repeats the same mistake. Your agent can't force-push, skip tests, or repeat a refactor you already rejected. No more fix-loops.

FAQ

Common questions

How is ThumbGate different from RLHF?
RLHF updates model weights. ThumbGate doesn't touch the model — it injects past feedback into context so your agent is conditioned by your corrections. Think of it as a behavioral immune system, not a training pipeline. The gate blocks are hard enforcement, not soft suggestions.
What's the tech stack?
SQLite+FTS5 lesson DB for fast full-text search. MemAlign-inspired dual recall (principle-based rules + episodic context). Thompson Sampling for adaptive gate sensitivity per failure domain. LanceDB + Apache Arrow for local vector search with Hugging Face embeddings. ContextFS for context assembly. Bayesian belief updates on each memory. PreToolUse hook enforcement blocks known-bad actions before execution. All local-first — no cloud required.
What AI agents and editors does this work with?
ThumbGate works with Claude Code, Cursor, Codex, Gemini CLI, Amp, OpenCode, and any other MCP-compatible agent. Cursor ships with a plugin bundle in this repo. Codex ships with a repo-local app plugin profile. VS Code works when you run an MCP-compatible agent inside it, but this repo does not ship a standalone VS Code extension today.
Do I need a cloud account?
No. Free gives you full local enforcement — captures, recalls, gates, and hook blocking on your machine. No cloud, no account, no signup. Pro adds a personal local dashboard on your machine plus DPO export. Team is the hosted rollout lane when your group wants a shared lesson database, org dashboard, and team-wide gate visibility.
How are gates different from prompt rules?
Prompt rules are suggestions that agents can ignore. Pre-Action Gates are enforcement — they block the action before execution. Gates are auto-generated from your feedback and use Thompson Sampling to adapt over time.

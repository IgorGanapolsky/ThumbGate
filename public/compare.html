Why ThumbGate Wins

  • Enforcement, not suggestions. Prompt rules in CLAUDE.md or .cursorrules are instructions the agent can ignore. ThumbGate intercepts tool calls at the PreToolUse hook level and physically blocks dangerous actions before they execute.
  • Learns and adapts automatically. Every thumbs-down becomes a data point. After repeated failures with the same pattern, ThumbGate auto-generates a prevention rule. Gates adapt their sensitivity over time using Thompson Sampling -- aggressive gates that over-block get tuned down, effective gates get reinforced.
  • Works across all major AI coding agents. One install command covers Claude Code, Cursor, Codex, Gemini CLI, Amp, and OpenCode. No per-agent configuration needed.
  • Prevention is cheaper than recovery. Manual code review catches mistakes after the PR. Git revert catches them after the push. ThumbGate catches them before the tool call executes -- saving the time, context switches, and cleanup cost of undoing damage.

Frequently Asked Questions

How do you prevent AI coding agents from making unsafe changes to shared repositories?

ThumbGate enforces approval policies via PreToolUse hooks that fire before every agent tool call. When an agent attempts a dangerous action — force-pushing to main, deleting production files, or bypassing CI — the hook intercepts and blocks it before execution. Teams configure shared enforcement rules in a central policy file, so every agent on every seat respects the same governance constraints.

What is agent governance for engineering teams?

Agent governance is the practice of enforcing shared policies, approval workflows, and audit trails across all AI coding agents operating on a shared codebase. ThumbGate provides agent governance at $99/seat/mo: shared enforcement memory, approval policies for protected actions, CI gate integration, and a full audit trail of every agent decision and human override.

How do you enforce approval policies for AI agent tool calls?

ThumbGate's gates engine lets teams define approval policies for specific tool calls — for example, requiring human approval before any agent pushes to a protected branch or modifies a production config. Policies are stored in shared config, enforced via PreToolUse hooks on every seat, and all decisions are logged to the audit trail.

What is the difference between CLAUDE.md rules and enforcement hooks?

CLAUDE.md rules are plain-text instructions the AI agent reads at session start. The agent can ignore, misinterpret, or forget them mid-session. Enforcement hooks (PreToolUse hooks) are code that runs in the Claude Code runtime before every tool call — they physically block execution and cannot be overridden by the agent. ThumbGate auto-generates enforcement hooks from feedback so teams get hard guarantees, not soft suggestions.

What is a pre-action gate?

A pre-action gate is an enforcement layer that intercepts AI agent tool calls before they execute. Unlike prompt rules that agents can ignore, pre-action gates physically block dangerous actions such as force-pushing to main, deleting production files, or committing code with failing tests.

Does ThumbGate work with Claude Code?

Yes. ThumbGate works with Claude Code, Cursor, Codex, Gemini CLI, Amp, OpenCode, and any MCP-compatible agent. Run npx thumbgate init --agent claude-code to auto-configure PreToolUse hooks.

This includes shared enforcement memory across all seats, approval policies for protected actions, CI gate integration, an org-level audit trail, and pilot rollout support. How is ThumbGate different from .cursorrules or CLAUDE.md rules?

Cursor rules and CLAUDE.md are prompt-level instructions that the AI agent can read, forget, or override. ThumbGate enforces rules at the tool-call level via PreToolUse hooks. When a tool call matches a known failure pattern, it is physically blocked before execution. Additionally, ThumbGate auto-generates prevention rules from feedback — no manual rule writing required.

Can ThumbGate learn from mistakes automatically?

Yes. When your AI agent makes a mistake, give a thumbs-down with context. After repeated failures with the same pattern, ThumbGate auto-generates a prevention rule and wires it as a pre-action gate. In team mode, lessons learned by one seat propagate to all seats. Gates adapt over time using Thompson Sampling, a Bayesian multi-armed bandit algorithm.

The quiet cost of agent migrations

Roo Code is a VS Code extension and its MCP wire format transfers to Cline directly. The visible part of the migration - swap the extension, keep the shortcuts - takes five minutes.

The hidden cost is memory. Every thumbs-down you gave Roo lived in Roo's context window and project memory: "don't git push --force to main," "this repo uses pnpm," "never auto-generate Prisma migrations on prod." When Roo goes dark on May 15, that institutional memory evaporates. You teach Cline the same mistakes from scratch.

Vendor-scoped lesson memory is a design mistake. In 2026 your agent's memory should outlive any single vendor.

What ThumbGate adds to Cline

ThumbGate stores lesson memory in a local SQLite file (.thumbgate/memory.sqlite) with FTS5 full-text search. Every thumbs-down becomes a row. When Cline proposes a tool call, ThumbGate's MCP server runs gate_check against the DB and blocks known-bad patterns before the call executes.

The DB lives in your project, not in Roo, not in Cline, not in our servers. When Cline eventually gets replaced in 2027, you copy one SQLite file and the memory moves with you.

One-command migration

npx thumbgate init --agent cline

The installer:

  • Registers the ThumbGate MCP server in Cline's VS Code globalStorage settings (cline_mcp_settings.json)
  • Copies .clinerules into your project so Cline knows when to call gate_check
  • Creates the local lesson DB at .thumbgate/memory.sqlite
  • Prints every file it touched so you can roll back

Any lessons you already captured under Roo live in .thumbgate/memory.sqlite and carry over unchanged.

What gets gated

The shipped .clinerules file tells Cline to call gate_check before:

  • execute_command that runs git push, git reset --hard, rm -rf, sudo, or curl ... | sh
  • Cloud mutations — aws, gcloud, az, railway, vercel deploy, gh release delete
  • write_to_file or replace_in_file targeting .env, *.pem, *.key, or files under .git/
  • browser_action that submits a form or clicks on a production URL

You can add or remove patterns by editing .clinerules.

Verify it is working

npx thumbgate verify --agent cline

Then inside Cline, ask it to run git push --force on a dummy branch. It should call thumbgate.gate_check first and refuse.

FAQ

When does Roo Code actually stop working?

The team announced the sunset on 2026-04-21. Final shutdown is 2026-05-15.

Does Cline use the same MCP format as Roo?

Yes. Any MCP servers you ran under Roo Code transfer to Cline verbatim.

Where does ThumbGate store my lessons?

In a local SQLite file at .thumbgate/memory.sqlite inside your project. No cloud, no account, nothing to orphan if a vendor sunsets again.

Does ThumbGate work if I go back to Claude Code or switch to Cursor?

Yes. The lesson DB is agent-agnostic. Running npx thumbgate init --agent <name> wires any supported agent against the same DB.

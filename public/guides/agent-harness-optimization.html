A better harness keeps global instructions lean, loads MCP schemas only when needed, and turns feedback into Pre-Action Gates. ThumbGate makes that workflow measurable and enforceable.

Why harnesses matter

The model is not the whole system. The harness decides which instructions, tools, context packs, approval rules, and verification gates the agent sees before it acts. When every rule lives in one global prompt, the agent burns attention on context it does not need.

ThumbGate keeps the harness in the useful zone: short discovery docs, progressive MCP tool loading, workflow-specific gate harnesses, and hard enforcement when a known-bad action tries to repeat.

What ThumbGate audits

  • Global AGENTS.md, CLAUDE.md, and GEMINI.md token pressure.
  • Progressive MCP discovery through lightweight tool indexes and schema URLs.
  • Specialized deploy, code-edit, and database-write gate harnesses.
  • Feedback-to-enforcement coverage through thumbs-down prevention rules and Pre-Action Gates.

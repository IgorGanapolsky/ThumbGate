Autoresearch Agent Safety for Self-Improving Coding Agents

4 min read · Designed for benchmark-search loops, CI agents, and coding automation

TL;DR: Autoresearch-style agents can search for better code, but the buying risk is trust. ThumbGate sells the missing control plane: gates that stop skipped holdout tests, fake proof, unsafe edits, and reward hacking before a loop promotes itself.

The wedge

Autoresearch turns coding agents into experiment runners. That is exciting because the loop can propose a patch, run a benchmark, inspect the result, and keep the improvement.

The risk is just as obvious: a loop that is rewarded for "better" can cheat the metric, overfit the benchmark, skip a holdout, hide a failure, or promote a variant without enough evidence. That is where ThumbGate belongs.

What ThumbGate gates

  • Promotion without the agreed primary and holdout checks.
  • Claims of improvement without commands, changed files, logs, and verification evidence.
  • Benchmark edits that make the score easier instead of the product better.
  • Risky filesystem, deploy, SQL, git, or secret-touching actions inside the experiment loop.
  • Repeated failures captured from thumbs-down feedback and promoted into prevention rules.
Buyer promise: Let the agent search. Do not let it grade itself without a Reliability Gateway.

Starter harnesses that convert

Start with the measurements teams already understand. Each one becomes a visible gate, not a vague promise.

  • npm test and coverage: block promotion when tests are skipped or coverage proof is missing.
  • Playwright duration: block a variant that improves one page while slowing the user path.
  • Bundle size: block growth without a signed exception and proof of value.
  • Lint and type checks: block "works on my branch" wins that fail shared quality bars.
  • CI failures: block merge or rollout when the exact run is still pending or red.

How to wire the pack

npx thumbgate init
npm run feedback:rules
npm run prove:automation

Use construct_context_pack with the autoresearch-brief template before experiment runs. The pack gives the agent compact memory of previous research failures, rules, and proof expectations.

Frequently Asked Questions

Why do Autoresearch-style agents need gates?

A self-improving loop can optimize the wrong signal, skip holdout tests, or promote a cherry-picked run. ThumbGate blocks known-bad promotion patterns before the agent accepts the variant.

What does ThumbGate add to an Autoresearch loop?

ThumbGate adds structured thumbs-up/down feedback, prevention rules, Thompson Sampling, ContextFS proof packs, and pre-action gates for risky experiment and promotion steps.

Can this work with normal CI checks?

Yes. Start with gates around npm test, lint, Playwright duration, bundle size, and CI failures. The same rules can later harden Team workflows.

## Course Title (Option A)

Workflow Hardening Sprint: Pre-Action Gates for AI Agents

## Course Subtitle

Turn one repeated Claude Code / Codex / Cursor / Gemini CLI failure into an enforceable prevention rule with proof.

## Who it’s for

- Solo operators and consultants shipping AI-assisted workflows
- Teams standardizing one risky workflow step (build/review/deploy/data)
- Anyone seeing repeat failures: unsafe tool calls, context drift, broken PR edits, bad deploys, credential mistakes

## Learning outcomes

- Identify the “repeat-failure seam” in a real workflow
- Define approval boundaries and blast radius
- Write a concrete Pre-Action Gate (what to block, what to require, how to prove)
- Produce a proof pack you can show a buyer or teammate

## Modules (draft)

1. The Reliability Gateway loop (feedback → memory → prevention rules → enforcement)
2. The workflow map (phases, ownership, invariants, “no-go” actions)
3. Failure taxonomy (context drift, permission mistakes, rollout regressions, bad diffs)
4. Pre-Action Gates (checks, prompts, invariants, deny/allow policies)
5. Proof runs (what to verify, how to prevent “looks good” hallucinations)
6. Escalation paths (human review, rollback plans, audit trails)
7. Next step decision tree (Diagnostic vs Sprint vs Pro vs nothing)
